# 슬라이드별 명세

> 각 슬라이드의 콘텐츠와 애니메이션을 정의한다. 구현의 단일 진실 소스.
> 상태 표시: 🟡 검토 중 · 🟢 확정 · ⚪ 미작성

## 슬라이드 1 — 인트로 (팀 Aice 등장) 🟢

**목적:** 팀 이름 Aice를 임팩트 있게 각인. "Aice = Ace + AI" 컨셉을 애니메이션으로 표현하고 팀원을 소개.

**애니메이션 시퀀스:**

1. **"Ace"** 가 가는 윤곽선(outline)으로 스르륵 떠오름 (각 글자 스태거)
2. 짧은 호흡
3. **A와 c 사이가 벌어짐** — 가운데 공간(slot)이 확장
4. 벌어진 틈으로 **"i"가 미끄러져 들어옴** (Ace 직후 곧바로 — 늦지 않게)
5. **클라이맥스** — "A"와 "i"만 안에서부터 흰색으로 **채워지고** 글로우 + 살짝 스케일. "c·e"는 윤곽선으로 남음
6. **상단 라벨("Team") · 태그라인 · 팀원 4명** 순차 공개
7. (예정) 다음 슬라이드로 전환 신호

**비주얼 톤 (확정):**
- 배경: **검정** (#0a0a0b) — 다크 시네마틱
- 글씨체: **Fraunces** 정자체 (워드마크), 본문/UI·한글은 **Pretendard**
- 색: **모노톤** (파란색·질감 그라데이션 없음)
- 강조 방식: **솔리드(채움) vs 윤곽선** 대비 — A·i 채움, c·e 윤곽선
- 모션: 튕김 없이 "스르륵" 흐르는 부드러운 이징(expo/power)

**팀원 (슬라이드 하단, 4열 깔끔 배치):**
| 역할 | 이름 | 전공 |
|---|---|---|
| 팀장 | 전승환 | 소프트웨어융합전공 |
| 팀원 | 문효은 | 환경공학전공 |
| 팀원 | 이승은 | 국어국문학과 |
| 팀원 | 조명원 | 에너지화공전공 |

**편집 가능 요소:** 타이밍, 라벨/태그라인 문구.

**상태:** 🟢 확정 · 코드 구현 완료
구현 위치: `index.html`(#slide-1 섹션) · `css/slide-1.css` · `js/slide-1.js`(`AiceSlide1`)

## 슬라이드 2 — 핵심 인사이트 "관리" 🟢

**목적:** 에너지 절약에는 수많은 요소가 필요하지만, 결국 끝까지 남아야 하는 것은 **관리**임을 영화처럼 각인.
서사: 여러 키워드가 모두 빛남 → 전등 꺼지듯 하나하나 소등 → 끝까지 "관리" 하나만 남아 크게 피어오름.
("불을 끈다"는 연출 자체가 에너지 절약 메타포 — 그 와중에 꺼지지 않고 남는 것이 관리.)

**콘텐츠:**

- 빛났다 꺼질 에너지 절약 키워드 20개 (솔리드 글로우 텍스트, Pretendard, 크기·밝기 차등 → 원근감):
  태양광 · 단열 · LED 조명 · 대기전력 · 고효율 가전 · 절전 모드 · 스마트 플러그 · 적정 온도 ·
  전기차 · 자연 채광 · 사용량 측정 · 피크 회피 · 인버터 · 에너지 등급 · 재생에너지 ·
  난방 효율 · 콘센트 차단 · 수요 반응 · 실시간 모니터링 · 행동 습관
- 히어로 단어 **"관리"** — Pretendard 800, 흰색 솔리드 + 글로우. 중앙에 고정, 처음엔 키워드 크기로
  축소되어 함께 빛나다가 끝에 크게 개화. 문구 없이 단어 하나.

**애니메이션 시퀀스:**

1. **점등** — 키워드들과 중앙의 "관리"가 화면 곳곳에서 하나둘 불을 켬(랜덤 스태거, 글로우 점등)
2. **호흡** — 모든 글자가 빛나는 상태로 잠시 머묾
3. **소등** — 전등 꺼지듯 키워드가 하나하나 깜빡 후 꺼짐(무작위 순서, 빨라졌다 마지막 몇 개는 감속).
   어둠이 깔리며 "관리"의 빛은 강해지고 배경 스포트라이트는 사그라듦
4. **정적** — "관리" 하나만 어둠 속에 작게 남음
5. **개화** — "관리"가 영화처럼 크게 피어오름(scale 0.18→1, 글로우 만개, 스포트라이트 다시 밝아짐)

**비주얼 톤:** 검정 배경 · 모노톤 · 흰빛 글로우 · expo/power 스무스 모션. 컬러 액센트 없음.
슬라이드 1 배경(스포트라이트·파티클·비네팅) 재사용.

**기술 메모:**

- 슬라이드 진입 시 1회 재생 — IntersectionObserver(≥55% 노출). 새 의존성 없음.
- 키워드는 JS가 5×4 그리드 + 지터로 흩뿌림(중앙은 "관리" 자리로 비움).
- 글로우는 `filter: drop-shadow`로 점등/소등. "관리"는 히어로 크기로 렌더 후 `scale`로 축소→개화(엣지 선명).
- 소등 점멸 간격은 가속 후 감속하는 곡선 — 마지막 몇 개가 천천히 꺼지며 긴장 고조.

**편집 가능 요소:** 키워드 목록(`js/slide-2.js`의 `KEYWORDS`), 타이밍.

**상태:** 🟢 확정 · 코드 구현 완료
구현 위치: `index.html`(#slide-2 섹션) · `css/slide-2.css` · `js/slide-2.js`(`AiceSlide2`)

## 슬라이드 3 — "측정" (피터 드러커 인용) 🟢

**목적:** 슬라이드 2의 *"관리"* 에서 → *"관리하려면 먼저 측정해야 한다"* 로 잇는 다리.
경영학의 아버지 피터 드러커의 명언으로 **측정 → 관리 → 개선**의 논리 사슬을 각인하고,
다음 챕터(제품 소개)에 키워드 **"측정"** 을 심는다.
슬라이드 2가 `관리` 한 단어를 남겼듯, 슬라이드 3은 `측정` 을 남긴다.

**레이아웃 — 좌우 분할:**
- **좌측(약 42%):** 흑백 인물 사진. `object-fit: cover`. 안쪽(우측) 가장자리는 검정 그라데이션으로
  페이드 → 분할선이 배경에 녹아듦. 사방을 옅게 비네팅해 다크 톤에 통합.
- **우측(약 58%):** 출처 라벨 · 히어로 인용구 · 킥커.
- 배경(`.bg` 스포트라이트·입자·비네팅)은 슬라이드 1·2와 동일하게 재사용(스포트라이트는 사진 쪽으로 치우침).

**콘텐츠 · 문구 (편집 가능):**
- **출처 라벨:** `PETER F. DRUCKER` / `경영학의 아버지` — 슬라이드 1 `Team` 라벨 톤. 아래 얇은 룰 라인.
- **히어로 인용구** (Pretendard, 본문은 흐리게 dim) — 슬라이드의 **메인**. 각 문장은 한 줄에 들어간다:
  > 측정할 수 없으면 관리할 수 없고,
  > 관리할 수 없으면 개선할 수 없다.
- **체인 키워드:** `측정 · 관리 · 개선` 만 솔리드 흰색 + 글로우 — "색이 아닌 형태 대비" 강조.
- 대본의 마무리 문장("…에너지 절약의 출발점입니다")은 발표자가 말로 전달하고, 슬라이드에는 별도 문구를 두지 않는다.

**애니메이션 시퀀스 (GSAP 타임라인, 약 6s):**

1. **현상** — 어둠 속에서 스포트라이트가 번지며 사진이 낮은 명암 → 정상으로 서서히 *인화*
   (opacity·brightness 상승, scale 1.06→1). 암실 인화 느낌, `expo/power`.
2. **라벨** — 우측 상단 출처 라벨 페이드인 + 룰 라인이 좌→우로 그어짐.
3. **인용구 본문** — 큰 따옴표와 줄이 단위로 스르륵 떠오름(line reveal). 키워드는 아직 흐림.
4. **체인 점화 (피날레)** — `측정` → `관리` → `개선` 순서로 한 단어씩 솔리드로 채워지며 글로우 점화
   (미세 scale 펄스). `관리` 점화는 슬라이드 2 솔리드 강조의 메아리(두 인스턴스 동시 점화).
   세 단어가 모두 켜진 채 슬라이드가 마무리된다.

**비주얼 톤:** 검정 배경 · 모노톤(흑백 사진+흰 텍스트) · 흰빛 글로우 · expo/power 스무스 모션.
컬러 액센트 없음. 슬라이드 1·2 배경 재사용.

**기술 메모:**

- 슬라이드 진입 시 1회 재생 — IntersectionObserver(≥55% 노출). R 키로 재생. 새 의존성 없음.
- 사진은 프로젝트 루트의 `peter.png`(로컬, 완전 오프라인). `<img>` 로드 실패 시 깨진 아이콘을 숨겨
  좌측 패널이 어둠으로 우아하게 대체되도록 처리.
- 사진 "현상"은 CSS `filter`(grayscale·brightness·contrast) + opacity·scale 을 GSAP로 트윈.
- 체인 키워드는 인라인 `<em class="kw">`(`display:inline-block`), 점화는 color + `drop-shadow` 글로우.

**편집 가능 요소:** 인용구 줄바꿈, 출처 라벨, 타이밍, 사진 파일.

**상태:** 🟢 확정 · 코드 구현 완료
구현 위치: `index.html`(#slide-3 섹션) · `css/slide-3.css` · `js/slide-3.js`(`AiceSlide3`)
사진 에셋: `peter.png` (프로젝트 루트 · 사용자 제공)

## 슬라이드 4 — 솔루션 (B-EMS → C-EMS) 🟢

**목적:** 슬라이드 3의 *"측정"* 에서 실제 **제품**으로 이어간다.
단일 건물 에너지 관리 시스템(B-EMS)을 출발점으로 삼아, 영남대학교 캠퍼스 전체를 아우르는
**C-EMS(캠퍼스 에너지 관리 시스템)** 로 확장되는 순간을 한 편의 줌아웃 카메라 무브로 보여준다.
슬라이드 2가 `관리`, 슬라이드 3이 `측정` 을 남겼듯 슬라이드 4는 제품명 **`C-EMS`** 를 남긴다.

**발표 대본 흐름:**

1. 화면 중앙의 단일 건물 관제 화면을 보며 `B-EMS`를 설명한다.
   - "단일 건물의 에너지를 관리하는 B-EMS"
   - "개별 건물의 전력 가계부 — 실시간 부하·층별 부하·효율"
   - "건물 하나에 적용되는 시스템을 영남대학교 캠퍼스 전체에 씌우면 어떨까?"
2. 발표자가 손을 양옆으로 펼치며 스페이스바(또는 →·PageDown)를 누른다.
3. 카메라가 하늘 위로 올라가는 듯 부감으로 줌아웃되고, 캠퍼스 건물 군집이 드러난다.
4. `B-EMS`가 `C-EMS`로 바뀌며 "캠퍼스 에너지 관리 시스템"으로 각인된다.

**서사 — 한 섹션 안의 2막 타임라인:**
- **1막 (B-EMS):** 어둠 속 중앙에 업라이트 빌딩 한 채가 떠오르고, 좌우에 관제 콘솔 패널이 켜진다.
  "한 건물의 에너지를 실시간으로 읽고 관리하는 대시보드"처럼 보여야 한다.
- **2막 (C-EMS):** 시점이 부감으로 줌아웃된다. 단일 건물은 캠퍼스 군집의 중심 코어가 되고,
  주변 건물에서 코어로 네트워크 연결선이 이어진다. `B`가 `C`로 크로스페이드되어 `C-EMS` 헤드라인으로 변신한다.

**렌더링 — 정사영(orthographic) 부감 아이소메트릭 (중요):**
- ⚠️ **빌딩은 절대 기울어 보이면 안 된다.** 원근(`perspective`)을 쓰면 높은 건물의 수직 모서리가
  위로 갈수록 모이며 "쓰러지는" 느낌이 난다. 그래서 **원근을 제거하고 정사영**으로 렌더한다.
- 회전은 정적 CSS 래퍼 `.tilt`(`rotateX(-54deg) rotateY(-36deg)` — rotateX가 바깥)가 고정으로 갖는다.
  GSAP은 `.world` 에 **스케일·세로이동만** 적용한다. → 수직 모서리가 항상 화면 수직선이 되어 안 기운다.
- ⚠️ **rotateX는 음수(-54°) — 위에서 내려다보는 부감(버드 아이).** 양수면 아래에서 올려다보게 되어
  윗면이 backface로 숨고 "두 면만 붙은 판"처럼 보인다. 음수 부감이라야 윗면이 드러나 직육면체로 읽힌다.
- 건물은 CSS 3D 직육면체 5면. 정사각형 바닥면 공유, 높이만 다르게. 정면은 밝게·우측면은 그늘지게·
  윗면은 가장 밝게 — 세 면의 명암을 분리해 입체감을 살린다.

**1막 — 단일 건물 관제 콘솔:**
- **히어로 빌딩:** DOM 창문 격자(점등/소등 셀, 일부 깜빡임), 옥상 HVAC 유닛, 바닥→옥상 에너지 펄스,
  발밑 글로우 디스크, 옅은 원근 격자 바닥.
- **좌측 패널 `REAL-TIME LOAD`:** `현재 전력`(0→128.4 kW 카운트업), `오늘 사용량`(1,842 kWh),
  `오늘 부하 곡선` SVG 스파크라인(좌→우 드로잉 + 피크 마커). 수치는 라벨 위 / 큰 숫자 아래(KPI 스타일).
- **우측 패널 `FLOOR LOAD`:** 층별 부하 막대(8F·6F·3F·B1 + kW 라벨, 0→목표 채움), 효율 게이지(SVG 반원 0→87%).
- 패널은 코너 브래킷으로 감싼 관제 화면 톤. 하단 중앙에 건물 프로필 카드(`건물 A-01`).
- **건물 콜아웃:** 빌딩 왼쪽 빈 공간(패널과 빌딩 사이)에서 노드 → 리더 선이 끌어나오고 라벨이 뜬다 —
  eyebrow `단일 건물` + `건물 한 채를 통째로 관리`. 1막 인트로 끝부분에 등장, 2막에선 HUD와 함께 사라짐.

**2막 — 캠퍼스 C-EMS:**
- 부감 줌아웃, 캠퍼스 건물 24채가 무작위 스태거로 팝인. 격자를 기준점으로 삼되 강한 지터(±46)로
  흩뿌려 실제 캠퍼스처럼 불규칙하게 배치 — 셀 수보다 적게 골라 빈터도 생긴다. 그리드 바닥 위 도시.
- 캠퍼스 건물마다 점등 창문 몇 개가 켜진다 — C-EMS가 확립될 때 도시가 하나둘 불을 켜며 살아난다.
- 중심 코어 링 + 코어에서 퍼지는 펄스 링(바닥으로 퍼졌다 사라짐, 반복) + 캠퍼스 건물 → 코어
  네트워크 연결선(선을 따라 빛 점이 흐름).
- 헤드라인 `B→C` 크로스페이드 → `C-EMS`, 부제 `캠퍼스 에너지 관리 시스템`,
  집계 스트립 `28개 동 · 4.2 MWh/일 · 실시간`.

**진행 방식 — 발표자 단계 제어:**
슬라이드 4는 두 막으로 나뉘고, 막 사이는 발표자가 **스페이스바(또는 →·PageDown)** 로 넘긴다.

## 슬라이드 5 — 예측 엔진 (머신러닝 작동 원리) 🟢

**목적:** 슬라이드 4에서 C-EMS 제품을 소개하고, 발표자가 **실제 Campus-EMS 앱으로 전환해 메인
대시보드 데모**(3D 지도·전력 예측 패널·건물 클릭)를 보여준 뒤 덱으로 돌아온다. 슬라이드 5는
데모에서 본 "전력 예측"이 **어떻게** 만들어지는지 — 머신러닝 학습 원리를 시각화한다.
슬라이드 2가 `관리`, 3이 `측정`, 4가 `C-EMS` 를 남겼듯 슬라이드 5는 **`예측`** 을 남긴다.
> 데모 화면 자체는 실제 앱이 담당하므로 덱에 재현하지 않는다. 이 슬라이드는 ML 원리 한 장.

**발표 대본 (정리본 — 슬라이드에 깔리는 내레이션):**
> 현재 예측 결과는 머신러닝 학습을 기반으로 합니다.
> 기온·습도·일사량 같은 **기상 데이터** 와 함께, 건물의 용도·시험기간·축제·휴일 여부,
> 그리고 과거 전력 패턴 같은 **캠퍼스 특성 데이터** 를 머신러닝으로 학습시켰습니다.
> 상황에 따라 각기 다른 변수들을 매칭해 학습시켰기에, 여러 조건에서도 그 상황에 맞는 예측이 가능합니다.

**레이아웃 — 가로 3단 수렴 파이프라인 (좌→우 흐름):**
- **좌 · 입력 변수 + 가중치:** 변수 두 묶음을 각각 **그룹 박스(모듈)**로 묶는다 — 그룹 라벨이
  박스 헤더(하단 구분선)가 되고 칩들이 박스 안에 담겨, 라벨이 칩들과 한 덩어리로 읽힌다.
  각 행은 `칩 + 적용 가중치`.
  - `기상 데이터` 박스 — 칩: 기온 · 습도 · 일사량 (3개)
  - `캠퍼스 특성 데이터` 박스 — 칩: 건물 용도 · 시험기간 · 축제 · 과거 전력 패턴 (4개)
  - 칩은 **윤곽선** pill(Pretendard). 칩 오른쪽에 그 입력에 적용된 **가중치 값**(`×0.86` 등)이
    빛 점이 모델에 닿을 때 표시된다. 연결선은 그룹 박스의 오른쪽 모서리에서 출발한다.
  - 변수 목록은 정리본 기준. 대본의 `휴일`은 시각 균형(3+4)을 위해 생략 — 꼭 다 넣지 않는다.
- **중앙 · 예측 모델 패널:** 코너 브래킷 패널(슬라이드 4 관제 콘솔 톤). 헤더 `예측 모델`,
  본문에 **가중치 행렬**(6×4=24 셀), 하단에 `모델 학습` 진행 막대(0→100%).
  학습 중 행렬 셀들이 난수 가중치로 churn(빠르게 회전)하다가 한 칸씩 **lock-in**(최종값 고정·솔리드)
  되며 모델이 완성된다. 모든 입력에서 패널로 가느다란 연결선이 뻗고, 빛 점이 모델로 흘러든다.
- **우 · 예측 전력 사용량:** 라벨 `예측 전력 사용량` + **큰 숫자 카운트업**(`0.0` → `142.8 kW`,
  감속 카운트) — 모델이 100% 완성되는 순간 차오른다. 아래 슬림 예측 곡선 + 적정 범위 밴드.
- 입력 칩은 윤곽선, 모델·출력은 솔리드 — 색이 아닌 **형태 대비**.

**텍스트 (최소):**
- 상단 eyebrow: `예측 엔진`
- 하단 킥커: `맥락을 읽는 예측` — `예측`만 솔리드 강조(대본 punchline "상황에 맞는 예측"의 응축).
- 그 외 텍스트는 그룹/칩/모델/출력 라벨뿐.

**애니메이션 시퀀스 (GSAP 자동재생 타임라인, ~9s):**
1. **eyebrow** `예측 엔진` 스르륵 등장 + 빈 모델 패널이 떠오름.
2. **기상 데이터** — 그룹 박스가 떠오르고 → 칩 3개가 스태거로 점등.
3. **캠퍼스 특성 데이터** — 그룹 박스가 떠오르고 → 칩 4개 스태거 점등.
4. **수렴 + 가중치 적용** — 각 입력에서 모델로 연결선 드로잉, 빛 점이 흘러들고,
   점이 닿을 때 칩 옆에 적용 가중치(`×0.86` …)가 뜬다.
5. **학습** — 모델 가중치 행렬이 난수로 churn → `LOCK_START`부터 한 칸씩 lock-in,
   `모델 학습` 막대가 0→100%로 차오른다.
6. **모델 완성** — 24셀이 모두 잠기고 학습 100% → 패널이 글로우(테두리 점등).
7. **예측 출력** — 모델 완성에 맞춰 모델→출력 연결선이 이어지고, **예측 전력 사용량이
   `0.0`→`142.8`로 카운트업**, 슬림 예측 곡선 + 적정 범위 밴드가 드로잉된다.
8. **마무리** — 킥커 `맥락을 읽는 예측` 솔리드 개화(`예측` 글로우 강조).
- 진입 후 앰비언트: 모델 패널 글로우가 은은히 호흡, 잠긴 행렬 셀 몇 개가 미세하게 깜빡.

**비주얼 톤:** 검정 배경 · 모노톤 · 흰빛 글로우 · `expo`/`power` 스무스 모션. 컬러 액센트 없음.
배경(`.bg` 스포트라이트·입자·비네팅) 슬라이드 1~4와 동일 재사용.

**기술 메모:**
- 순수 CSS + 인라인 SVG(연결선·예측 곡선) + GSAP. **새 의존성 없음**, 완전 오프라인.
- 연결선은 화면 좌표 기반 — JS가 `getBoundingClientRect`로 가중치·모델·출력 위치를 계산해
  **직선** SVG path 생성(`stroke-dashoffset` 드로잉). 빛 점은 `attr:{cx,cy}` 보간 — GSAP
  코어 AttrPlugin만 사용, MotionPathPlugin 의존 없음.
- 가중치 행렬 churn은 churn 트윈의 `onUpdate`가 3프레임마다 미잠금 셀 텍스트를 난수로 교체,
  lock-in은 `tl.call`로 셀별 최종값 고정. 카운트업은 프록시 트윈 `onUpdate`(슬라이드 4 `addCount` 패턴).
- 슬라이드 진입 시 1회 자동재생 — IntersectionObserver(≥55% 노출). R 키로 재생.
- 폰트 로드 후(`document.fonts.ready`) 레이아웃 확정 뒤 연결선 좌표 계산. 리사이즈 시 재계산.
- 슬라이드 추가 규칙 준수: `index.html`(#slide-5) · `css/slide-5.css`(#slide-5 스코프) ·
  `js/slide-5.js`(`AiceSlide5`) · `main.js` 연결. 클래식 `<script>`(ES 모듈 금지).

**편집 가능 요소:** 입력 가중치(`js/slide-5.js`의 `INPUT_WEIGHTS`), 예측 사용량 값(`OUT_TARGET`),
행렬 셀 수(`MATRIX_N`), eyebrow·킥커 문구, 예측 곡선 데이터(`buildForecast`), 타이밍 상수.

**상태:** 🟢 코드 구현 완료 · 검증 통과 (사용자 시각 확인 대기)
구현 위치: `index.html`(#slide-5 섹션) · `css/slide-5.css` · `js/slide-5.js`(`AiceSlide5`)
