## 작업 방식 (사용자와 합의된 흐름)

- **슬라이드를 한 장씩** 사용자와 논의하며 만든다. 한 번에 전부 만들지 않는다.
- 각 슬라이드: 디자인 논의 → 사용자 확인 → 코드 구현 → `docs/` 갱신.
- 시각 피드백이 필요하면 미리보기 서버 사용: `python -m http.server` 또는 `.claude/launch.json`의 `aice`.
