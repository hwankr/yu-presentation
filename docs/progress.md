## 작업 방식 (사용자와 합의된 흐름)

- **슬라이드를 한 장씩** 사용자와 논의하며 만든다. 한 번에 전부 만들지 않는다.
- 각 슬라이드: 디자인 논의 → 사용자 확인 → 코드 구현 → `docs/` 갱신.
- 시각 피드백이 필요하면 미리보기 서버 사용: `python -m http.server` 또는 `.claude/launch.json`의 `aice`.

## 비고

- 본문/UI 폰트: 당초 Inter+Noto Sans KR → **Pretendard** 단일 폰트로 정리 (한글+영문 커버, 단일 파일).
- 미리보기(백그라운드 탭)에서는 rAF 정지로 자동재생이 안 됨 — 실제 브라우저(포그라운드)에서는 정상.
  슬라이드 2 검증은 GSAP 타임라인을 `seek()`로 프레임 이동시켜 컴퓨티드 스타일로 확인했다.
- 거부된 디자인 방향(파란색·메탈릭 질감·bounce 이징)은 `AGENTS.md`의 "하지 말 것" 참고.
- 슬라이드 2 키워드 목록은 `js/slide-2.js`의 `KEYWORDS` 배열에서 바로 편집 가능.
- 슬라이드 4 캠퍼스는 매 재생마다 무작위 배치 — 건물 수·격자는 `js/slide-4.js`의 `buildCity`,
  카메라 각도·스케일·타이밍은 같은 파일 상단 상수(`ACT1/ACT2_*`)와 `build` 타임라인에서 조정.
